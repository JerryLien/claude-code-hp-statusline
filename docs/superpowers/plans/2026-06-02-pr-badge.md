# PR Badge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show the open PR for the current branch in the statusline row1 as `🔀#1234✓` (rpg) / `🌷#1234✓` (bloom), state-coloured, with an OSC 8 clickable link.

**Architecture:** Add three values (`pr.number` / `pr.url` / `pr.review_state`) to the single `python3 -c` parse block, emit them as shell vars, then render a badge in row1 right after the worktree/branch block. Wrap the badge in an OSC 8 hyperlink when `pr.url` is present, and teach the row width-measurement helper to strip OSC 8 so the hidden URL never inflates the wrap calculation.

**Tech Stack:** Bash + embedded python3 (no jq), existing `tests/statusline.test.sh` harness (mock JSON on stdin, `grep -qF` assertions).

### Task 3: Strip OSC 8 in width measurement

**Files:**
- Modify: `statusline-hp.sh` (`dw()` python in the responsive block)
- Test: `tests/statusline.test.sh`

**Why:** The width-measurement python receives the row strings BEFORE `echo -e`, so escapes are the literal 4-char `\033`. Its `ANSI_RE` only strips SGR (`\033[...m`); the OSC 8 sequence (with the full URL inside) is currently counted as visible width, which can wrongly wrap a line that actually fits.

- [ ] **Step 1: Write the failing test**

Append to `tests/statusline.test.sh` before `# --- Summary ---`:

# A ~150-char URL would blow past COLUMNS=80 if counted, forcing a wrap.
# With correct stripping, the visible row1 (⚔ Opus 🔀#1234✓) + row2 (ctx bar)
# easily fits on one line at 80 cols.
PR_LONGURL='{"model":{"display_name":"Opus"},"pr":{"number":1234,"review_state":"approved","url":"https://github.com/an-extremely-long-organisation-name-here/an-extremely-long-repository-name-goes-right-here/pull/1234567890123456"}}'

assert_single_line "pr-osc8-width-stripped" "80" "rpg" "$PR_LONGURL"
assert_single_line "pr-osc8-width-stripped-bloom" "80" "bloom" "$PR_LONGURL"

- [ ] **Step 3: Add OSC8_RE to dw()**

In `statusline-hp.sh`, find the width-measure python inside the responsive block:

```python
import sys, re, unicodedata
# Match both actual ESC byte (\x1b) and literal backslash-0-3-3 (\033)
ANSI_RE = re.compile(r"(?:\x1b|\\033)\[[0-9;]*m")
def dw(s):
    s = ANSI_RE.sub("", s)
```

Replace those lines with:

```python
import sys, re, unicodedata
# Match both actual ESC byte (\x1b) and literal backslash-0-3-3 (\033)
OSC8_RE = re.compile(r"(?:\x1b|\\033)\]8;;.*?(?:\x1b|\\033)\\")
ANSI_RE = re.compile(r"(?:\x1b|\\033)\[[0-9;]*m")
def dw(s):
    s = OSC8_RE.sub("", s)
    s = ANSI_RE.sub("", s)
```

Note: `OSC8_RE` strips `ESC]8;;<url>ESC\` (open) and `ESC]8;;ESC\` (close). It uses `]8;;` (right bracket) so it never touches SGR `[...m` (left bracket). It runs BEFORE `ANSI_RE` so the colour codes left inside the link still get stripped afterwards.
